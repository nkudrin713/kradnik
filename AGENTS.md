# Agent Instructions

## Communication

- Отвечай на языке полученного сообщения.
- Отвечай максимально кратко и по делу. Экономь токены.
- Пиши просто, без вежливых вступлений, повторов и лишних слов.
- Не добавляй длинные объяснения, планы или резюме, если задача простая.
- Сначала результат, потом короткое пояснение, только если нужно.
- Не описывай очевидные действия и базовые вещи, если пользователь не попросил.
- Для простых задач пиши 1-3 коротких предложения.
- Для ошибок укажи причину и минимальное исправление.

## Code Changes

- Делай только то, что пользователь прямо попросил.
- Если нужно внести изменение вне прямого запроса, сначала спроси.
- Не улучшай архитектуру, стиль, названия, форматирование или поведение без запроса.
- Не трогай соседний код без необходимости.
- Не делай масштабный рефакторинг без отдельного разрешения.
- Следуй существующим паттернам проекта.
- Не удаляй и не переписывай чужие незакоммиченные изменения без прямого разрешения.
- Для изменений в коде коротко перечисляй, что изменено.

## Dependencies

- Не добавляй новые зависимости без явной необходимости.
- Перед добавлением зависимости кратко объясни, зачем она нужна.
- Никогда не используй `latest`.
- Всегда указывай конкретную версию.
- Версия должна быть достаточно свежей и стабильной.

## Kotlin

- Не используй wildcard import.
- Запрещены импорты вида `import something.*`.
- Всегда импортируй конкретные классы и функции.
- Сохраняй существующий стиль Kotlin/Spring кода.

## Tests And Build

- Не запускай сборку и юнит-тесты после каждого действия.
- Если изменение маленькое и локальное, не запускай тесты автоматически.
- Запускай сборку и тесты перед коммитом или пушем.
- Запускай сборку и тесты, если накопилось много изменений в git index.
- Запускай сборку и тесты после масштабных переработок или рефакторингов.
- Для полного прогона проекта используй:

```bash
./gradlew test
```

- Если тесты не удалось запустить, укажи причину и риск.

## Git

- Не выполняй destructive-команды вроде `git reset --hard`, `git checkout --` или массового удаления без явного запроса.
- Не создавай коммиты, если пользователь прямо не попросил.
- В конце работы сообщай, какие файлы изменены.
